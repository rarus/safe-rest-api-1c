Регистр сведений «Журнал конфликтов HTTP-запросов»

Регистр сведений «Журнал конфликтов HTTP-запросов» предназначен для записи и хранения информации о конфликтах и блокировках, возникших при обработке входящих HTTP-запросов. 

Информация о конфликтах представлена в удобном для чтения и анализа виде.

Сведения о конфликтах хранятся в разрезе сервиса, даты блокировки и ключа идемпотентности входящего HTTP-запроса.

В форме записи регистра есть возможность перейти к исходной записи входящего HTTP-запроса в регистре сведений "Журнал входящих HTTP-запросов", или разблокировать конфликт вручную.

Предусмотрено автоматическое разблокирование конфликтных HTTP-запросов по расписанию с помощью регламентного задания "Разблокирование конфликтов HTTP-запросов".

Ключевые параметры разблокирования устанавливаются для каждого сервиса отдельно в настройках справочника "Сервисы". 

Каждый параметр разблокирования является частью стратегии, котрую можно подобрать опытным путем в пользовательском режиме под ситуацию конкретного сервиса, без дополнительного программирования.

Срок хранения записей журнала для каждого сервиса задается в настройках справочника "Сервисы". 

Устаревшие записи журнала удаляются по расписанию регламентного задания "Сокращение журналов входящих HTTP-запросов и конфликтов HTTP-запросов".
